# 📡 FrameLab (شبیه‌ساز لایه پیوند داده)

**FrameLab** یک ابزار آموزشی و تعاملی تحت وب است که به‌منظور شبیه‌سازی مکانیزم‌های اصلی **لایه پیوند داده (Data Link Layer)** در شبکه‌های کامپیوتری طراحی و پیاده‌سازی شده است.

این پروژه با هدف درک بهتر مفاهیم انتقال داده، تشخیص خطا و کنترل جریان، سه مبحث مهم این لایه را به‌صورت بصری و قابل تعامل شبیه‌سازی می‌کند:

- Bit Stuffing / Destuffing
- CRC (Cyclic Redundancy Check)
- Stop-and-Wait ARQ

---

# ✨ امکانات پروژه

## ۱. بیت‌استافینگ و دی‌استافینگ (Bit Stuffing / Destuffing)

- دریافت رشته باینری تا حداکثر **۱۰۰ بیت**
- درج خودکار بیت `0` پس از مشاهده **۵ بیت `1` متوالی**
- بازیابی خودکار داده در سمت گیرنده (Destuffing)
- بررسی صحت داده بازیابی‌شده نسبت به داده اولیه
- نمایش نتیجه عملیات به‌صورت مرحله‌به‌مرحله

---


## ۲. تشخیص خطا با CRC (Cyclic Redundancy Check)

- دریافت داده باینری تا **۳۲ بیت**
- دریافت چندجمله‌ای مولد (Generator)
- محاسبه باقی‌مانده تقسیم دودویی (XOR Division)
- تولید خودکار مقدار CRC
- تولید Codeword نهایی
- امکان ویرایش دستی Codeword
- قابلیت **تزریق خطای تصادفی (Random Error Injection)**
- بررسی Codeword در سمت گیرنده و تشخیص وجود یا عدم وجود خطا

---

## ۳. شبیه‌سازی Stop-and-Wait ARQ

- ارسال فریم‌ها با شماره توالی متناوب `0` و `1`
- شبیه‌سازی دریافت ACK
- اعمال تأخیر زمانی مشابه انتقال واقعی
- شبیه‌سازی تصادفی گم‌شدن Frame یا ACK
- مدیریت Timeout
- ارسال مجدد (Retransmission) تا دریافت موفق ACK
- نمایش تمامی رویدادها در بخش Log

# 🚀 نحوه اجرا

ابتدا مخزن پروژه را Clone کنید:

```bash
git clone https://github.com/your-username/FrameLab.git
```

سپس وارد پوشه پروژه شوید:

```bash
cd FrameLab
```

از آنجایی که این پروژه با **Vite** توسعه داده شده است، ابتدا وابستگی‌ها را نصب کنید:

```bash
npm install
```

یا به اختصار:

```bash
npm i
```

سپس سرور توسعه را اجرا کنید:

```bash
npm run dev
```

پس از اجرای دستور فوق، آدرس نمایش داده‌شده در ترمینال (معمولاً `http://localhost:5173`) را در مرورگر باز کنید.

> **نکته:** این پروژه یک برنامه مبتنی بر **Vite** است؛ بنابراین برای اجرای صحیح آن، استفاده از `npm install` و `npm run dev` الزامی است و باز کردن مستقیم فایل `index.html` در مرورگر توصیه نمی‌شود.

# 📖 راهنمای استفاده

## سناریوی اول: Bit Stuffing

1. در بخش اول برنامه رشته‌ای شامل چندین بیت `1` متوالی وارد کنید.

نمونه:

```text
10111111110
```

2. روی دکمه **اعمال Bit Stuffing** کلیک کنید.

3. مشاهده خواهید کرد که پس از هر پنج بیت `1` متوالی، یک بیت `0` به‌صورت خودکار اضافه می‌شود.

4. سپس روی **بررسی Destuffing** کلیک کنید.

5. در صورت صحت عملیات، رشته اولیه بازیابی شده و پیام موفقیت نمایش داده می‌شود.

---

## سناریوی دوم: CRC

### حالت بدون خطا

مقادیر زیر را وارد کنید:

```text
Data      : 1001010111
Generator : 111011
```

سپس:

- روی **محاسبه CRC و Codeword** کلیک کنید.
- مقدار CRC و Codeword تولید می‌شود.
- روی **بررسی Codeword** کلیک کنید.

در این حالت پیام زیر نمایش داده می‌شود:

```text
بدون خطا
Remainder: 00000
```

---

### حالت دارای خطا

بدون تغییر داده‌ها:

- روی **تزریق خطای تصادفی** کلیک کنید.

یا

- یکی از بیت‌های Codeword را به‌صورت دستی تغییر دهید.

سپس مجدداً روی **بررسی Codeword** کلیک کنید.

در این حالت برنامه وجود خطا را تشخیص داده و باقی‌مانده تقسیم غیرصفر نمایش داده می‌شود.

---

## سناریوی سوم: Stop-and-Wait ARQ

1. روی **شروع شبیه‌سازی** کلیک کنید.

2. برنامه شروع به ارسال چهار فریم متوالی می‌کند.

3. شماره توالی فریم‌ها بین `0` و `1` تغییر می‌کند.

4. تمامی رویدادها در بخش **Log** نمایش داده می‌شوند.

در صورت گم‌شدن Frame یا ACK، پیام‌های زیر مشاهده خواهند شد:

- Timeout
- Retransmission
- دریافت موفق ACK

این فرآیند تا تحویل موفق داده ادامه پیدا می‌کند.
